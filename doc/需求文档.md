## 1. 项目概述

### 1.1 背景

随身 WiFi 场景下常见运营商限速、商家流量虚标、信号与环境导致的网速问题。本产品为 **Web 端工具**，通过「信息检测 → 网速诊断 → 去除限速」引导用户完成排查与处理；当前阶段检测与测速由 **前端实现**，接口层预留对接真实后端。

### 1.2 目标

- 技术栈：**Vue 3**（`<script setup>`）+ **Vite** + **TypeScript** + **Pinia** + **Element Plus**，图表使用 **ECharts**。
- 布局：**左侧主流程** + **右侧实时控制台**；窄屏下控制台以 **底部抽屉** 打开。
- 流程闭环：完成设备侧信息展示、测速与诊断结论、一键去限速及去限速后的 **再次测速验证** 引导。

### 1.3 用户角色

- **终端用户**：使用随身 WiFi、需要自助排查网速与限速相关问题的消费者。

## 2. 范围、术语与原则

### 2.1 展示原则（信息检测页 / 首页步骤）

**仅展示**与「已连接设备」相关的：

- **硬件信息**（如设备型号）；
- **购买套餐**（套餐名称、套餐内宣传高速流量等对外信息）；
- **网络参数**（制式、信号、运营商、连接设备数等）。

**不在信息检测页展示**（避免误导或暴露内部实现细节）：

- 实际高速上限、流量虚标比例、累计当月流量、剩余高速流量、当前限速状态等（若业务需要，由诊断结论或配置说明承接，而非首页指标卡）。

### 2.2 术语

| 术语 | 含义 |
|------|------|
| 信息检测 | 步骤一：用户主动触发检测，拉取设备与套餐相关展示数据。 |
| 网速诊断 | 步骤二：测速（约 5 秒）+ 输出诊断结论与建议。 |
| 去除限速 / 去控 | 步骤三：调用解除限速接口；交互上展示分阶段去控过程，成功后展示本次诊断数据并引导前往步骤二验证。 |
| 应用配置 | 通过「应用配置」页可调整检测与诊断相关行为。 |

## 3. 整体布局与交互

### 3.1 布局结构

- **顶栏**：应用标题、帮助入口（含跳转 **应用配置**）。
- **主区左侧**：步骤进度（三步）、当前步骤标题、步骤内容、底部「上一步 / 下一步」。
- **主区右侧（宽屏）**：实时控制台（时间戳日志、清空等）。
- **窄屏**：浮动按钮打开控制台抽屉。

### 3.2 实时控制台

- 记录关键操作与结果（检测开始/完成、测速、诊断结论、去限速请求等）。
- 日志条数上限（如 200 条），超出丢弃最旧记录。
- 支持清空日志。

### 3.3 全局交互约定

- **进入应用**：不自动做设备检测；仅提示就绪，由用户在步骤一点击「开始检测 / 重新检测」。
- **保存配置**：不自动触发检测；需用户回主页手动检测。
- **步骤切换**：主内容区使用过渡动画切换步骤组件。

## 4. 主流程需求（三步）

### 4.1 步骤一：设备信息检测

**目的**：确认已连接设备，并展示对外可见的套餐与网络侧信息。

**展示内容（实现对齐）**

- 套餐区：购买套餐名称、套餐内高速流量（宣传值，单位 GB）。
- 指标卡：设备型号、网络制式（含 LTE 文案）、信号强度（仪表盘 + dBm + 等级文案）、运营商、连接设备数。

**交互**

- 「开始检测 / 重新检测」：请求设备信息；进行中骨架屏；完成后可进入下一步。
- 不在此步骤展示限速程序、虚标比例、剩余流量、累计流量、限速状态等（见 §2.1）。

### 4.2 步骤二：网速诊断

**目的**：完成测速并给出诊断结论，为是否「去限速」提供依据。

**测速与展示**

- 点击「开始诊断」：进度条约 5 秒，完成后展示下行（ECharts 仪表盘）、上行/延迟/抖动及对应阈值说明。
- 展示诊断结论 `el-alert`（严重度配色）：结论 + 建议文案。

**诊断逻辑（实现对齐）**

- 结论优先受 **设备检测阶段给出的 `limitedStatus`**（应用配置中的「限速原因」）影响，再结合测速指标与「去限速成功后再次测速」等标志综合判断（详见 `computeDiagnosis`）。
- 典型结论类型包括：**网络正常**、**信号弱**、**运营商限速**、**商家流量虚标导致提前触发运营商限速**、**网络异常**等。
- **商家虚标**场景下，虚标比例可在 **诊断建议文案** 中体现（配置项 `vendorVirtualPercent`）；不要求单独横幅展示（当前产品已去掉独立「商家流量虚标」提示条与「限速程序检测」提示条）。

**去限速成功后再次测速**

- 若用户已完成去限速且进入验证测速，诊断可按配置优先给出「网络正常」类结论（`afterSuccessfulRemoveLimit` + 指标达标）。

### 4.3 步骤三：一键去除限速

**目的**：在诊断允许时执行解除限速，并引导用户回到步骤二验证。

**启用条件**

- 当诊断结论为 **运营商限速** 或 **商家流量虚标导致提前触发运营商限速** 时，允许「一键去除限速」；否则按钮禁用并提示原因。

**去控过程（动态反馈）**

- 点击后进入 **分阶段去控**：步骤条（如连接通道 → 扫描策略 → 执行去控 → 同步侧）+ 条纹进度；与接口并行，保证最短展示时长，避免一闪而过。

**去控成功后**

- **不自动跳转**回步骤二；先在步骤三展示 **成功结果区**：
  - 展示 **本次（去控前）诊断数据**：诊断结论、建议，以及上下行/延迟/抖动与阈值对照。
  - 提供主按钮 **「前往网速诊断验证」**：跳转到步骤二并 **清空本次测速与诊断状态**，便于重新测速；成功后测速按产品规则可 **强制指标达标**（用于验证闭环）。
- 操作记录表：时间、方式（接口/自助）、结果、说明。

**其他**

- 「自助指南」弹窗：话术与一键复制。
- 「重新诊断」：回到步骤二并清空测速结论（不替代上述「验证」主路径的完整定义，以当前实现为准）。

## 5. 应用配置

路径：通过界面进入 **应用配置** 页面（具体入口以顶栏「帮助」为准）。

**配置项（实现对齐）**

| 分类 | 字段含义 |
|------|----------|
| 套餐 | 套餐名称、宣传高速流量（GB）、实际高速流量上限（GB，用于测速归一化，首页不展示该「实际」值） |
| 设备回填 | 设备型号、网络制式、运营商、信号 dBm 范围、连接设备数范围 |
| 诊断 | 限速原因（`limitCause`）、商家虚标比例（%）、弱信号阈值（dBm） |

已移除的、不影响主流程闭环的配置（如剩余流量生成方式、达量触发余量 UI、限速状态灯阈值等）不再写入需求正文；内部常量由前端 `src/api/mock.ts` 维护。

## 6. 数据模型与接口

### 6.1 核心类型（与代码一致）

`DeviceInfo` 除基础字段外，包含：`totalTrafficGB`、`remainingHighspeedGB`（内部计算用）、`virtualRatioPercent`、`limitProgramDetected`、`limitedStatus`（`normal` | `signal_weak` | `operator_limit` | `vendor_virtual_limit`）等。

`SpeedTestResult`：`downloadMbps`、`uploadMbps`、`latencyMs`、`jitterMs`。

`Diagnosis`：`conclusion`、`severity`（`info` | `warning` | `error`）、`suggestion`。

`RemoveLimitRecord`：`timestamp`、`method`（`api` | `guide`）、`success`、`message`。

### 6.2 接口（概念）

| 能力 | 说明 |
|------|------|
| 设备信息 | 延迟约 500ms，按配置生成 `DeviceInfo`。 |
| 测速 | 延迟与 UI 进度对齐（约 5s），结果与设备状态、实际上限等逻辑一致。 |
| 去除限速 | 当前产品可为 **固定成功** 以保障流程闭环，成功后可标记「去限速已成功」并影响后续测速行为。 |

具体函数名与参数以 `src/api/mock.ts` 为准。

## 7. 非功能需求

- **性能**：首屏可接受范围内加载；测速过程不阻塞界面交互（异步）。
- **日志**：控制台日志有上限，避免内存无限增长。
- **响应式**：宽屏双栏；窄屏控制台抽屉；主流程单栏可读。
- **可替换性**：API 层与 Store 分离，便于替换为真实 HTTP / WebSocket。

## 8. 界面与动效（与实现对齐）

- **步骤进度条**：三步，与 `STEP.DEVICE / SPEED / REMOVE` 一致。
- **步骤一**：套餐条 + 指标卡网格；卡片带入场延迟动画。
- **步骤二**：测速进度条（条纹流动）、仪表盘、诊断 `el-alert`。
- **步骤三**：去控步骤条 + 进度、成功快照区、按钮组与记录表。
- **控制台**：日志列表、级别颜色、清空。

未在代码中统一实现的项（如全局按钮 Ripple、控制台单条复制）不写入强验收；若后续补充，再更新本节。

---

## 9. 验收标准

- **布局**：宽屏左右分栏、窄屏抽屉控制台可用；步骤切换正常。
- **信息检测**：仅展示 §2.1 / §4.1 约定内容；检测由用户点击触发，非进入即自动检测。
- **网速诊断**：测速流程完整，结论与建议展示正确；阈值与 `limitedStatus` 逻辑符合当前 `diagnosis` 实现。
- **去限速**：禁用/启用与诊断结论一致；去控过程可见；成功后展示诊断数据与验证入口；记录可查阅。
- **配置**：保存后行为符合 §5；不自动触发检测。
- **构建**：工程可正常生产构建（TypeScript 与打包无错误）。
